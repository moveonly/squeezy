Prefer `decl_search`, `reference_search`, or `symbol_context` for symbol-shaped queries in Rust, Python, Java, C#, Go, C/C++, and JavaScript/TypeScript files when the query is a bare name and the languages have nominal type systems (Rust, Go, C#, C++, Java) — they follow imports, qualified paths, and re-exports that regex misses. For Python and JS/TS aliased-import resolution, or any multi-symbol tabulation (count usages across N symbols, aggregate fields per class), `read_file` with a focused query or script is usually one shot. Use `read_file` for literal text (strings, comments, config keys) or files outside Rust, Python, Java, C#, Go, C/C++, and JavaScript/TypeScript. Read a bounded byte slice from one workspace file and return its sha256 receipt. Use `read_file` once the graph (or a free-form `grep`) has produced a path and span.
