# Build-script allowlist (F15-pi)
#
# Every transitive Cargo dependency that ships a `build.rs` (and therefore
# runs arbitrary code on every `cargo build`) must be listed here before
# `scripts/check_build_scripts.py` will accept the workspace. This mirrors
# pnpm's `allowedInstallScriptPackages` guardrail: a fresh transitive that
# silently picks up a build script in a new release is treated as a CI
# failure until a human audits the script.
#
# Workspace-local crates (members of this Cargo workspace) are not gated by
# this file — `cargo` already surfaces those build scripts during regular
# code review.
#
# Format:
#   - One package name per line.
#   - Optional `name@version` to pin to an exact version. Bare `name`
#     accepts every version that resolves into the lockfile.
#   - `#` starts a line comment. Blank lines are ignored.
#
# When a new entry is needed:
#   1. Read the build.rs of the crate (and any helper modules it `include!`s).
#   2. Confirm the script only does benign work — emitting `cargo:` directives,
#      generating Rust sources from local templates, linking known system
#      libraries, etc. Reject anything that downloads, executes arbitrary
#      binaries from the network, writes outside `OUT_DIR`, or shells out to
#      install global state.
#   3. Add the bare crate name here (keep the file alphabetised within each
#      section) and reference the audit reason in the PR description.
